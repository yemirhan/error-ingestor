# Dashboard

The Error Ingestor dashboard provides a visual interface for viewing and analyzing errors captured from your applications.

## Features

- **Error List** - Browse and search captured errors
- **Error Details** - View full error information including stack traces
- **Trends** - Visualize error patterns over time
- **Filtering** - Filter by app, error code, user, and time range
- **Code Distribution** - See which errors occur most frequently

## Running the Dashboard

### Development

```bash
pnpm dev --filter dashboard
```

Dashboard runs at http://localhost:5173

### Production Build

```bash
pnpm build --filter dashboard
pnpm --filter dashboard preview
```

## Pages

### Error List

The main page showing all captured errors.

**Features:**
- Sortable columns
- Pagination
- Quick search
- Time range selection

**Columns:**
| Column | Description |
|--------|-------------|
| Code | Error code (e.g., NETWORK_ERROR) |
| Message | Error message text |
| Platform | ios, android, or web |
| User | User ID if available |
| Time | When the error occurred |

### Error Details

Click any error to see full details:

- **Error Info**: Code, message, app version
- **Stack Trace**: Full stack trace with syntax highlighting
- **Metadata**: Custom metadata as JSON
- **Tags**: Key-value tags
- **Platform**: Platform and version info
- **User**: User ID and tracking info

### Trends

Visualize error patterns:

- **Time-Series Chart**: Error count over time
- **Granularity**: Switch between hourly and daily views
- **Metrics**:
  - Total errors
  - Unique error codes
  - Affected users

### Code Distribution

See which errors occur most frequently:

- Pie or bar chart of error codes
- Count per code
- Percentage breakdown

## Filtering

### By Application

Select an application from the dropdown to view only its errors.

### By Error Code

Filter to show only specific error codes:

```
Code: NETWORK_ERROR
```

### By User

Filter errors for a specific user:

```
User ID: user-123
```

### By Time Range

Select predefined ranges or custom dates:

- Last hour
- Last 24 hours
- Last 7 days
- Last 30 days
- Custom range

## Data Fetching

### Hooks

The dashboard uses custom hooks for data fetching:

```tsx
// Fetch errors with filters
const { data, loading, error } = useErrors({
  appId: "my-app",
  startTime: "2024-01-01",
  endTime: "2024-01-15",
  limit: 100,
});

// Fetch trends
const { data: trends } = useTrends({
  appId: "my-app",
  granularity: "day",
});

// Fetch error codes
const { data: codes } = useErrorCodes("my-app");
```

### API Client

```typescript
// src/lib/api.ts
export const api = {
  errors: {
    list: (params: ErrorsParams) =>
      fetch(`${API_BASE}/dashboard/api/errors?${toQueryString(params)}`),

    get: (id: string, appId: string) =>
      fetch(`${API_BASE}/dashboard/api/errors/${id}?appId=${appId}`),
  },

  trends: (params: TrendsParams) =>
    fetch(`${API_BASE}/dashboard/api/trends?${toQueryString(params)}`),

  codes: (appId: string) =>
    fetch(`${API_BASE}/dashboard/api/codes?appId=${appId}`),
};
```

```dockerfile

COPY . .

RUN pnpm build --filter dashboard

FROM nginx:alpine
COPY --from=builder /app/apps/dashboard/dist /usr/share/nginx/html
EXPOSE 80
```

## Security Considerations

The dashboard endpoints are currently unprotected. For production:

1. **Add Authentication**: Implement login/session management
2. **Protect Routes**: Add middleware to verify sessions
3. **CORS**: Restrict to dashboard domain only
4. **HTTPS**: Always use HTTPS in production

### Adding Basic Auth

```typescript
// Dashboard auth middleware
app.use("/dashboard/*", async (c, next) => {
  const session = c.req.header("Authorization");
  if (!validateSession(session)) {
    return c.json({ error: "Unauthorized" }, 401);
  }
  await next();
});
```
